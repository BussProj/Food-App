# Security Policy

## Supported Versions
Only the latest major version of the platform receives active security updates and patch releases.

| Version | Supported |
| --- | --- |
| 1.x.x | :white_check_mark: Yes |
| < 1.0.0 | :x: No |

## Reporting a Vulnerability
We take the security of our platform seriously. If you find a security vulnerability, please do **NOT** open a public issue on GitHub. Instead, report it through our secure disclosure protocol.

### Disclosure Protocol
1.  Send a detailed description of the vulnerability, including steps to reproduce, impact, and affected components, to **security@platform.com**.
2.  Do not publish details of the vulnerability until we have had a reasonable amount of time to analyze and fix the issue.
3.  We will acknowledge your report within 48 hours and provide a timeline for validation and patching.

### Scope
This policy covers the main application repository, container infrastructure templates, and official integration adapters. It does not cover individual third-party API availability or local physical printing hardware.
